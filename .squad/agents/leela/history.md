- 2026-05-20T09:59:21.307+02:00: Dependency posture audit: the crate has 7 direct runtime dependencies, 164 packages in `Cargo.lock`, and 96 unique packages in the normal dependency tree. `reqwest` is the dominant cost center (88-package subtree) and is forced into the default experience by `src/client.rs` (`ExceptionlessClient<T = HttpTransport>`, `with_api_key`) plus `src/config.rs` using `reqwest::Url`; secondary public dependency leaks are `serde_json::Value` in builder `.data(...)` methods (`src/error.rs`, `src/log.rs`, `src/feature.rs`) and `chrono::DateTime<Utc>` in `src/wire/event.rs`. Hard constraints for any reduction plan live in `README.md`, `src/lib.rs`, `examples/config_custom.rs`, `tests/regression_submission_path.rs`, and the stack-trace tests, all of which currently promise default HTTP transport, tokio-based async examples, and real captured stack traces.
- 2026-05-20T09:59:21.307+02:00: After the owner chose `lean_core_opt_in_http`, the next packaging comparison favors a same-crate, default-disabled `http` feature over a companion crate for the first cut. It eliminates default HTTP pollution immediately while preserving one crate name, one docs surface, and a reversible path; a companion transport crate better isolates alternate transports long-term, but adds release/docs coordination and renaming friction before the core boundary is stable.
